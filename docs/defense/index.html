《霞客行》游戏 Demo 开发记录

本页用于答辩时展示项目从策划收束、MVP 锁定、插件环境搭建到功能实现的过程证据。 后续每完成一个系统切片，都在这里补充改动、截图、验证命令和演示入口。

项目定位

核心体验

城镇接任务，小地图闯关，回合制战斗，通关后点亮大地图新区。

Demo 范围

先做一条完整闭环：青山驿站 → 黄山险径 → Boss → 解锁潭江渡口。

技术路线

UE 5.8、Paper2D/PaperZD 角色表现、UE MCP 自动化、C++/UMG 优先。

MVP 已锁定 节点小地图 回合制战斗 背包交易 NPC 任务

当前改动记录

2026-07-01

读取策划案并锁定可玩 MVP

从《霞客行》31 页工作版策划案和讨论记录中，收束为“城镇 Hub + 任务入口 + 杀戮尖塔式小地图 + 回合制战斗 + 大地图解锁”的单关闭环。

2026-07-01

迁移 TestMap 小地图原型参考资源

将 TestMap UE 5.4 的 Content/1Game 原型内容保留原包路径迁入本项目，作为杀戮尖塔式小地图 UI 与图标参考；项目默认地图、GameMode 与 MVP 规则仍由新写系统负责，不把该原型蓝图作为运行时权威数据来源。

2026-07-01

装配 GameXXK 内容目录和基础地图

通过 UE MCP 项目脚本创建 /Game/GameXXK 下的 UI、Maps、Characters、Data、Sprites 目录，生成 L_Main 与 L_QingshanInn 两张基础地图，并将默认启动地图指向 L_Main。

2026-07-02

角色源 Sprite Sheet 入库

将已选的深蓝高马尾主角、蓝衣随从和青绿商人 8 方向行走 sheet 复制到 Content/GameXXK/Sprites/Generated，并通过 UE MCP 导入为 Texture2D 资产，避免运行时依赖聊天软件缓存路径。

2026-07-02

主角 8 向 Paper2D/PaperZD 行走资产

分析深蓝高马尾 1024 行走图后，按 6 列 5 行切分，并用确定性水平镜像展开为 8 向 sheet；通过 UE MCP 创建 48 个 PaperSprite、8 个 PaperFlipbook、AS_Hero_Flipbook、ABP_Hero_PaperZD 和 PZD_Hero_Walk_8Dir directional 序列，城镇 AGameXXKTownPlayerPawn 默认加载 FB_Hero_Walk_South，确保 Play 模式进入城镇时主角有可见 Paper2D 表现，并给后续 PaperZD 方向动画接入留下资产链路。

2026-07-02

城镇主角 8 向行走表现切换

为 AGameXXKTownPlayerPawn 增加 8 向朝向状态和方向 Flipbook 表，WASD/方向键/轴输入按下、组合与释放时会切换到对应 FB_Hero_Walk_North、East、SouthWest 等资源；停止移动后保留最后朝向，混用 WASD 与方向键时单个按键释放不会丢失仍按住的方向，自动化覆盖按键组合、轴输入和资源路径。

2026-07-02

青山驿站基础地形与光照

参考 Ocean 项目的场景自动化方式，但不引入 Water 或地图生成；新增 GameXXKEditor 固定 Landscape 自动化，在 L_QingshanInn 中生成 16 组件基础地形，并放置道路、边界、地牢入口标记、PlayerStart、Directional Light、SkyLight、SkyAtmosphere 和雾效，写入地图 GameMode override。

2026-07-02

城镇出口 F 交互

新增 AGameXXKTownExitActor，放入青山驿站地牢门位置；玩家走近后按 F，未接任务时保持城镇状态并提示先接任务，接任务后进入固定小地图流程。

2026-07-02

真实 UMG 主菜单与大地图入口

新增 UGameXXKPlayableRootWidget，由 AGameXXKMVPHUD 在 Play 模式创建并加入视口；根控件提供 Start/Continue、青山驿站和潭江渡口按钮，直接调用 MVP Subsystem 而不是 Canvas HUD 命令。自动化 GameXXK.MVP.PIE.MainMenuContinueWorldMap 覆盖真实 UMG 入口、未解锁区域禁用、Start 进入大地图和点击青山进入城镇。

2026-07-01

补齐 Play 模式原生可试玩入口

新增 MVP GameMode、PlayerController 与可点击 HUD，把主菜单、大地图选城镇、城镇任务/交易/补给、小地图节点、战斗胜负和 Boss 解锁串成编辑器 Play 模式可操作闭环；试玩脚本新增 PlayableShell 自动化覆盖。

2026-07-01

Playable HUD 自动存档

将 Play 模式 HUD 的 Start、选城镇、接任务、交易、小地图节点、战斗胜负等可演示命令接入 SaveGame 写入；Boss 通关后重新读取同一 slot 可恢复任务完成状态和潭江区域解锁。

2026-07-01

打通城镇 NPC 的 F 键交互

为城镇 NPC 增加交互范围：玩家进入范围后自动聚焦，按 F 可接任务并激活跟随，商人 NPC 可执行 MVP 一键购买补给；对应自动化覆盖 overlap 聚焦、离开清焦点和规则状态变化。

2026-07-01

城镇 F 交互自动存档

任务 NPC 和商人 NPC 的真实 F 交互成功后写入默认 SaveGame slot；自动化验证接任务后可重载出 Accepted 状态与跟随状态，买药后可重载出已扣除金币。

2026-07-01

接入插件底座

启用 Paper2D、PaperZD、ModelContextProtocol、MCPClientToolset、AllToolsets、EditorScriptingUtilities。PaperZD 使用 UE 5.8 已安装的 Marketplace 插件；编辑器自动化统一改走 UE 5.8 内置 MCP。

2026-07-01

建立自动化与答辩记录

加入 GameXXK 专用 scripts/ue_mcp_client.py、scripts/ue_mcp_smoke.py、scripts/ue_tdd_pipeline.py 和项目 MCP Toolset，本页作为后续答辩记录主页。

2026-07-01

新增 UGameXXKMVPRules 和 GameXXK.MVP.FullFlow 自动化测试，覆盖主菜单、大地图、城镇任务、跟随、交易、小地图节点、速度排序战斗、失败回城、补给重试、Boss 解锁和存档恢复；scripts/gamexxk_mvp_playtest.py 可一键运行该流程。

城镇交互 Shell

新增 AGameXXKTownPlayerPawn、UGameXXKInteractionComponent、AGameXXKTownNpcActor，验证 WASD/方向键自由移动、F 交互、任务 NPC、商人 NPC 和跟随 NPC 的 C++ 行为边界。

2026-07-01

UMG 连接层

新增 UGameXXKMVPSubsystem 与主菜单、大地图、任务弹窗、背包、交易、角色面板、小地图、战斗等 C++ Widget Base，验证 UI 按钮可驱动现有 MVP 规则 API。

验证记录

每次提交前补充最新验证命令和结果。当前建议验证入口：

  • python scripts/gamexxk_mvp_playtest.py：编译并运行 GameXXK.MVP 自动化试玩流程。
  • python scripts/gamexxk_content_assembly_check.py：通过 UE MCP 验证 /Game/GameXXK 内容目录、基础地图和默认地图配置。
  • python scripts/gamexxk_town_scene_check.py：通过 UE MCP 验证青山驿站固定 Landscape 地形、边界、入口交互 Actor、PlayerStart、基础光照、无 Water Actor 和地图 GameMode。
  • python scripts/gamexxk_sprite_source_check.py：通过 UE MCP 验证角色源 sheet PNG 和导入的 Texture2D 资产。
  • python scripts/gamexxk_character_visual_check.py：通过 UE MCP 验证主角 8 向 walk PaperSprite、PaperFlipbook、source UV、帧顺序和默认城镇 flipbook。
  • python scripts/gamexxk_paperzd_character_check.py：通过 UE MCP 验证主角 PaperZD AnimationSource_Flipbook、AnimBP 和 8 向 directional walk sequence。
  • python scripts/ue_tdd_pipeline.py --pie-duration 5：启动编辑器、等待 UE MCP、运行 PIE 并收集 [TDD] 日志。
  • python scripts/ue_mcp_smoke.py：连接正在运行的 UE MCP server，验证 meta tools、Toolsets、项目 GameXXKTDDToolset 和 GameFeatureData 配置。
  • git diff --check：提交前检查空白和补丁格式。
