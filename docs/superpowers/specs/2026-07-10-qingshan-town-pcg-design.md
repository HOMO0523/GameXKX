# 青山镇 QuickRoad / PCG 生成链路设计

日期：2026-07-10
状态：已完成交互设计确认，等待书面规格复核

## 1. 目标与边界

本阶段先跑通一条可重复、可调试、可扩展的实体城镇 PCG 链路。城镇采用“A：水门主轴型”布局，保留现有北门 F 任务交互与存档链路，不修改用户已经手调的关卡、角色、PaperZD、相机或 HD2D 参数。

首版追求结构正确、风格统一和性能可控，不追求一次铺满整座城镇。首版限制为 12–16 栋建筑和低密度植被；所有数量、面数、贴图、实例密度与裁剪距离必须被记录为可调整参数，后续依据实际性能数据扩容或降级。

## 2. QuickRoad 能力边界

QuickRoad 负责：

- 城镇范围与布局范围样条。
- 主路、支路、交叉口和道路网格。
- RoadEdge、Intersection 等道路侧数据。
- 道路贴合地形和必要的静态网格烘焙。

项目 PCG 负责：

- 河流、水面和河岸禁建区。
- 固定桥梁及桥头玩法区域。
- 房屋地块、建筑选择和入口朝向。
- 树木、灌木、石块、围栏及小型装饰。
- 固定玩法锚点和所有 PCG 排除区。

QuickRoad 中内部命名为 RiverTool 的功能实际上用于提取道路布局数据，不作为河流生成方案。插件未提供可直接使用的桥梁、河流或树木生成系统。

## 3. 空间骨架

城镇试作范围约为 240m × 180m，使用相对布局约束，最终绝对坐标以现有关卡北门 F 为基准：

- 北门 F 位于北侧中央，为固定锚点，不参与随机生成。
- 8m 宽主街从北门向南贯穿城镇。
- 4–5m 宽支路连接住宅、商铺和河岸区域。
- 集市位于主街中段、桥北侧，建议占地约 40m × 35m。
- 河流横穿城镇南侧约三分之一位置，宽 18–24m。
- 主桥位于主街与河流的交点，桥宽约 9m，为固定实体模型。
- 房屋位于主街两侧和集市外围，建筑边界距离道路 2–3m。
- 树木集中于城镇外围、河岸和房屋后院。

固定排除区包括北门 F、任务 NPC、集市中心、桥面、桥头、道路、路口和河岸安全带。固定锚点的优先级高于任何程序化内容。

玩家动线固定为：北门 F → 主街 → 集市 → 主桥 → 南岸码头。该动线也是首版 PCG 的主要视觉和可玩性验收路线。

## 4. 建筑资产集

首批制作六款实体建筑，保持白墙、深色木构和项目水墨卡通语言，通过屋顶轮廓、颜色、高度和附属结构形成差异：

| 类型 | 屋顶款式 | 主色 | 目标四边面预算 |
|---|---|---:|---:|
| 民居 A | 双坡悬山顶 | 青灰 | 20k–30k |
| 民居 B | 四坡歇山顶 | 靛蓝 | 约 30k |
| 沿街商铺 | 前檐双坡顶 | 墨绿 | 30k–40k |
| 客栈 / 酒楼 | 两层组合屋顶 | 暖红褐 | 最高 50k |
| 河岸仓房 | 低矮硬山顶 | 深灰 | 20k–30k |
| 桥头地标 | 错层飞檐顶 | 青蓝与铜绿 | 最高 50k |

六类建筑首版至少各出现一次；同款建筑不得沿街连续出现超过两栋。PCG 可以改变旋转、轻微比例和材质实例色差，但不得随机改变入口与道路的关系。

## 5. 建筑生成 JSON 与生产链路

每一款新建筑必须先建立独立 JSON，JSON 是该资产的生产来源记录，至少包含：

- `asset_id`、用途、版本和状态。
- 项目风格锁定描述及禁止项。
- 建筑尺寸、层数、占地边界和入口方向。
- 屋顶款式、屋顶颜色、墙体、木构、门窗和檐口规则。
- GPT 参考图生成提示和已选参考图路径。
- Tripo 高精度模型生成参数。
- 重拓扑目标四边面数和结果统计。
- 贴图分辨率及 Albedo、Normal、Roughness、AO 等可用贴图记录。
- UE 导入比例、轴向、Nanite、碰撞、材质和 Toon Profile 设置。
- PCG 分类、占地尺寸、入口 Socket / 朝向、权重和允许区域。
- 性能档位、回退资产和验证结果。

生产顺序固定为：

1. 编写并校验建筑 JSON。
2. 使用已确认的项目原画风格生成参考图，先确认屋顶、比例和配色。
3. 在 Tripo 生成高精度模型，不直接用低面数智能网格作为最终建筑。
4. 使用重拓扑功能将模型控制到 JSON 指定的四边面预算。
5. 在重拓扑结果上生成并检查贴图。
6. 导入 UE 5.8，校验尺寸、朝向、Pivot、UV、法线和简单碰撞。
7. 启用 Nanite，接入共享的 Substrate Toon BSDF 主材质和 Toon Profile。
8. 添加 PCG 元数据并进入候选资产集。

建筑参考图和贴图不得依赖生成式中文招牌文字，以避免乱码。需要文字的招牌由 UE 内独立贴花或材质层完成。

## 6. PCG 数据流

生成顺序固定如下：

1. 读取城镇边界、河流样条和固定玩法锚点。
2. QuickRoad 生成主路、支路、路口和 RoadEdge。
3. 从 RoadEdge 向外构造建筑候选带，并生成候选地块。
4. 从候选地块中扣除道路、路口、河岸、桥头、北门 F、任务 NPC 和集市禁生区。
5. 按地块占地尺寸、区域标签和权重选择六类建筑，入口朝向最近道路。
6. 对建筑地块执行边界、间距和重叠检查，不合格地块直接跳过。
7. 在剩余区域生成院落、小路、围栏、树木、灌木和石块。
8. 输出道路、河流、建筑候选区、有效地块、植被区和禁生区调试颜色。

固定随机种子必须得到可重复结果。桥梁、北门、任务 NPC 和集市中心不依赖随机种子。

## 7. 失败与回退策略

- 地块放不下建筑时跳过，不缩放到破坏比例，也不侵入排除区。
- 某类建筑缺失时使用相同占地等级的备用资产；无合格备用资产时保留空地。
- 桥梁无法正确连接两岸道路时停止该层生成并显示错误标记，不继续生成桥头内容。
- 资产 JSON、模型、材质或贴图缺失时，该资产不得进入正式 PCG 候选集。
- PCG 数据不可用时保留固定玩法锚点和现有关卡内容，不覆盖已有摆放。
- 所有生成操作先进入专用原型目录和测试关卡，验证后再决定是否集成到正式关卡。

## 8. 性能设计与可调档位

最低目标为中端 PC：RTX 2060 / RX 6600 级别、6–8GB 显存、16GB 内存，1080p 正常游玩目标不低于 60 FPS。

首版采用保守档位：

- 建筑总量 12–16 栋，硬上限 16。
- 普通建筑 20k–40k 四边面，只有客栈和桥头地标允许达到 50k。
- 每栋建筑优先 1 个材质槽，最多 2 个。
- 普通建筑使用 2K 贴图；只允许客栈和桥头地标使用 4K。
- 六款建筑共享 Toon 主材质与 Toon Profile，通过材质实例改变屋顶和木构颜色。
- 重复建筑、树木、灌木和石块使用 ISM / HISM，不为每个实例创建独立 Actor。
- 建筑仅使用简单盒体或少量凸包碰撞，禁止默认复杂模型碰撞。
- 小型杂物裁剪距离 40–60m，灌木 60–100m，树木 120–180m。
- PCG 默认在编辑器生成并缓存，运行时不重新生成整座城镇。
- 河流、道路和桥梁控制样条分段数量，远处关闭高成本装饰。

性能参数必须集中记录在一个城镇 PCG 性能配置中，不散落为不可追踪的图节点常量。至少记录：建筑上限、各类别权重、面数预算、纹理预算、植被密度、裁剪距离、碰撞策略、样条分段和运行时生成开关。

后续扩容顺序为：先测量首版 → 增加建筑到 18–24 栋 → 提高植被密度 → 添加装饰。性能超标时降级顺序为：降低植被密度 → 合并材质 / 减少材质槽 → 缩短裁剪距离 → 降低普通建筑面数；地标建筑最后降级。

## 9. 验收标准

### 布局与玩法

- 固定随机种子下布局可重复生成。
- 北门 F、任务 NPC、集市和主桥位置始终不变。
- 主桥正确连接两岸道路，不悬空、不插入河床。
- 房屋入口朝路，建筑不压道路、不进入河岸且互不穿插。
- 树木不出现在道路、建筑、桥面和交互区域。
- 玩家可从北门沿主街到集市、过桥到码头。
- 原有北门 F 接任务、跟随状态和手动存档链路不受影响。

### 资产与风格

- 六类建筑均有完整 JSON、参考图、高精度源模型、重拓扑结果和贴图记录。
- 六类建筑至少各出现一次，同款不连续超过两栋。
- 屋顶颜色和款式有清晰差异，但白墙木构和 Toon 表现保持统一。
- 所有模型完成比例、Pivot、法线、UV、碰撞、Nanite 和 Toon BSDF 检查。

### 性能与调试

- 首版建筑数量不超过 16，所有运行实例符合性能配置。
- 在目标硬件 1080p 典型游玩视角下达到 60 FPS 目标。
- 使用 `Stat Unit`、`Stat GPU`、Draw Call、显存统计和 PCG 生成耗时记录结果。
- PCG 提供道路、河流、建筑候选区、植被区和禁生区调试视图。
- 性能不达标时按照既定降级顺序调整，并将每次参数与结果回写到性能记录中。

## 10. 实施拆分

该设计后续实施应拆为以下顺序，避免一次性修改正式城镇：

1. QuickRoad 5.8 插件接入和演示关卡验证。
2. 原型城镇边界、道路、河流、桥梁和固定锚点。
3. 建筑 JSON 模板与性能配置。
4. 六款建筑的参考图、Tripo、重拓扑、贴图和 UE 导入。
5. 建筑地块与生成 PCG。
6. 树木、灌木和装饰 PCG。
7. 玩法链路、视觉和性能验收。

在原型验收前，不将程序生成结果覆盖到现有正式关卡。
