# 双页签任务面板设计

## 目标

在城镇 HUD 中加入一个与腾讯文档参考一致的水墨米纸任务面板。首版只有 **主线** 与 **支线** 两个可点击页签；不创建日常、奇遇的页签、资源或占位数据。

面板不是一张整图：框体与水墨笔触保持为高分辨率贴图，任务内容与数值由 UMG 实时绘制。这样新增任务、更新进度或切换语言都不需要重新切原画。

## 范围与非目标

### 首版范围

- HUD 的任务入口可打开和关闭任务面板。
- 主线、支线两页签均可点击，并按任务类别过滤当前列表。
- 任务卡显示标题、说明、奖励、进度、状态和行动按钮。
- `追踪` 在 HUD 中标记当前任务；再次点击可取消追踪。
- `前往` 只在已有的世界目标/地图入口可达时启用；不能到达时显示明确的不可用状态，不传送角色、不伪造完成。
- 城镇现有任务 NPC 的 `F → 确认对话框 → 接取` 流程仍是主线任务的唯一接取入口。

### 非目标

- 不新增日常或奇遇内容。
- 不把参考图中的任务标题、说明、进度或奖励数字烙进位图。
- 不修改用户已调节的角色相机、PaperZD、关卡摆放或材质。
- 不实现自动寻路、传送或新的任务剧情；这些需要独立设计。

## 视觉与资源边界

所有源图来自腾讯文档参考。每个小裁图完成后，先用生图功能按原画风格重绘/放大，再导入 UE；不把低分辨率截图直接放大到界面尺寸。重绘仅用于修复分辨率和边缘，不改变文字、色相或结构。

| 原图资源 | UE 纹理 | 用途 | 文本策略 |
| --- | --- | --- | --- |
| 任务窗框（空白） | `T_TaskPanelFrame` | 可拉伸的整窗框 | 无文本 |
| 返回箭头 | `T_TaskPanelBackArrow` | 顶部返回 | 固定切图 |
| 标题“任务” | `T_TaskPanelTitle` | 顶部固定标题 | 固定切图 |
| 选中页签底 | `T_TaskTabSelected` | 主线或支线当前状态 | 无文本 |
| 未选中页签底 | `T_TaskTabNormal` | 非当前页签 | 无文本 |
| “主线”“支线” | `T_TaskTabLabelMain`、`T_TaskTabLabelSide` | 页签标签 | 固定切图 |
| 任务卡底 | `T_TaskEntryFrame` | 重复任务条 | 无文本 |
| 金色行动按钮底 | `T_TaskActionTrack` | 追踪 / 取消追踪 | 文本实时绘制 |
| 青灰行动按钮底 | `T_TaskActionGo` | 前往 | 文本实时绘制 |
| 金币、EXP、令牌 | `T_RewardCoin`、`T_RewardExp`、`T_RewardToken` | 奖励行 | 图标切图，数值实时绘制 |

每个资源导入时均设置 UI 纹理压缩、`NoMipmaps`、`NeverStream` 和 sRGB。框体、任务卡和按钮必须预留干净边缘，使用 9-slice/Box Brush 拉伸；不可按普通 Image 整体缩放，以免边框变粗或文字区变形。

## 界面结构

```
任务面板（模态层）
├─ 纸本窗框
├─ 顶栏：返回箭头 + 固定“任务”标题
├─ 左栏：主线页签、支线页签
└─ 列表区
   └─ 任务卡 × N
      ├─ 实时：标题、说明、进度
      ├─ 奖励：图标贴图 + 实时数值
      └─ 行动：追踪/取消追踪 或 前往/不可用
```

任务卡在窗口内纵向滚动。主线和支线复用同一张任务卡底；选中状态只影响左栏页签、列表过滤和行动按钮状态，不复制两套窗口。

## 数据与交互

每条任务在 UI 层以只读视图数据表达：`Id`、`Category`（Main/Side）、`Title`、`Description`、`ProgressCurrent`、`ProgressTarget`、`Rewards`、`State`、`CanNavigate`。

- 主线数据从现有 `FGameXXKRuntimeState.QuestState` 映射：未接取时指向城镇任务 NPC；已接取时指向既有城镇出口/路线地图；完成时展示完成状态。
- 支线页签始终可点击。没有有效支线数据时显示一张同风格的“暂无支线委托”空状态卡，不虚构可领取任务。
- 打开面板默认恢复上一次选择的页签；第一次打开默认主线。
- 点击主线/支线立即重新筛选列表，并保持面板打开。
- 点击追踪只改变当前追踪任务 ID，刷新 HUD 目标提示；不改变任务进度。
- 点击前往仅调用已存在的可达目标入口。若任务没有目标，按钮禁用并保留原因；任何禁用动作不得修改存档或任务状态。
- 返回箭头和 Esc 仅关闭面板并恢复城镇输入；不会影响对话框、背包或任务状态。

## 验收

1. 在城镇 HUD 打开任务面板，窗框、页签、卡片、图标与按钮均使用导入的高分辨率资源，没有低清模糊或整图拉伸。
2. 只有主线和支线两个页签；二者都可点击并切换过滤结果。
3. 任务标题、说明、进度和数值不是贴图，修改运行时状态后会刷新。
4. 当前主线不经由任务面板直接接取；仍必须在世界中对任务 NPC 按 F，再在确认对话框点击接取。
5. 追踪、取消追踪、可用前往、不可用前往、Esc/返回均有确定结果，且不破坏已有存档、NPC 跟随和路线入口。
6. 自动化测试覆盖类别过滤、状态映射、追踪状态和禁用前往；PIE 覆盖打开/关闭、切换两个页签及任务 NPC 原有接取链路。

## 实施顺序

1. 从参考图按上表裁切 13 个原子资源，并逐个用生图功能放大后导入。
2. 建立数据模型和主线状态映射；支线采用可点击空状态，直到有真实支线内容。
3. 实现一个任务面板与两页签过滤，接入 HUD 打开/关闭。
4. 接入追踪与已有可达入口，明确禁用状态。
5. 自动化与 PIE 验收；重点检查高分资源、按钮命中和不破坏 NPC 接取流程。
