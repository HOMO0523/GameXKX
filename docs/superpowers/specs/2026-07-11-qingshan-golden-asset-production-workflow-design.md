# 青山镇黄金样板生产流程设计

日期：2026-07-11
状态：用户已批准采用方案 B，等待书面规格复核

## 1. 目标

把青山镇环境生产从“每条反馈触发一次新图、每个小步骤都请求确认”改为“黄金样板锁定、批量派生、三次验收”。用户只负责方向和三个关键门点；代理负责维护规则、生产、自检、淘汰失败版本、更新 JSON 和记录决定。

本流程首先覆盖实体建筑，随后以同样的门点方式扩展到桥、自然物、Paper2D 植物和 UE PCG 场景。建筑优先使用 Blender 精确建模；Tripo 只用于不要求精确接口的岩石和自然道具候选；植物与轻量小道具按既定 Paper2D 流程生产。

## 2. 现有流程问题

- 风格、结构、面数、拆件和场景布局在同一次图像反馈中混合，单次修改会意外改变多个变量。
- 对每个门窗、柱头或屋檐都请求确认，把内部判断成本转移给用户。
- 连续生图缺少稳定的参数基线，同类问题会在下一版本重新出现。
- 氛围图、Tripo 输入图、结构图和 UE 验收图承担了彼此冲突的用途。
- 原有商铺已经具有可用风格语言，却没有被提升为可量化的建筑母版。

## 3. 决策与覆盖范围

本文件是建筑生产流程的操作权威。它只覆盖 `2026-07-10-qingshan-environment-concept-json-production-design.md` 中与新建筑逐图审批、Batch 0 建筑解锁以及建筑 Tripo 输入有关的条款；地表、水体、山体、Paper2D 和道具的分类规则继续有效。

建筑风格权威改为：

1. 已有 `BLD_QS_L_A_MARKET_SHOP` 的整体歪扭、粗框门窗和手工搭建感。
2. 用户在本任务中确认的“大轮廓歪扭、粗门窗、粗梁柱、少细瓦、少斗拱、可拆件”规则。
3. 黄金建筑 `BLD_QS_M_A_INN` 通过 Gate 1 后的 `.blend`、固定渲染和参数 JSON。

旧风格板保留为色彩、墨线和环境关系参考，不再单独阻塞建筑黄金样板的 Blender 生产。

## 4. 黄金建筑视觉契约

黄金建筑为 `BLD_QS_M_A_INN`。它是建筑族的结构和风格母版，不是所有房屋共用的外形模板。

### 4.1 大形与歪扭

- 可见大轮廓采用 5%–10% 的受控非对称：屋脊轻弯、左右檐口高度不同、墙体轻微收分、主柱允许约 4–7 度视觉倾斜。
- 歪扭必须发生在屋顶、墙体、梁柱、门窗等大构件，不通过碎石、噪点、小木片或密集装饰制造“手工感”。
- 建筑必须保持单一稳定地面接触平面；Pivot、门口通行面、PCG 占地和模块 Socket 使用规整坐标。
- 左右视觉重量不同，但不得出现融化、坍塌、穿插或无法解释的受力关系。

### 4.2 门窗与木构

- 门窗是主要表情构件。框宽占开口宽度约 8%–12%，内部最多 2–3 个宽分隔，不使用密集格栅。
- 主入口采用宽双开门，整体允许略微偏离正中；相邻窗允许轻微梯形和高低错位。
- 柱头只保留一个主木托，结构确有需要时最多增加一个次级块；禁止三层以上重复斗拱。
- 阳台下只使用少量大支撑，禁止连续小方块和密集短梁。

### 4.3 屋顶与细节预算

- 屋顶首先是一个大色块和清楚剪影；不逐片建瓦、不用密集瓦线表达材质。
- 单栋建筑在 128 px 缩略图中必须读清，最多 3 个主形体和 5 组次级内部线。
- 招牌、灯笼、雨棚、额外屋檐装饰和可替换窗板均为独立模块，不焊死在主体中。
- 原画和模型不包含文字；文字由 UE 材质、贴花或独立牌面处理。

### 4.4 建筑之间的变化

批量建筑不能只更换屋顶颜色。每栋至少在以下五项中改变三项：

- 功能轮廓：客栈、沿街商铺、民居、工坊、河岸小屋。
- 高宽比例和层数。
- 屋顶方向、坡度、檐口偏压和附属体组合。
- 门窗数量、开口尺度与入口位置。
- 石基高度、侧棚、阳台或临河平台。

共享的是设计语法、材质和模块接口，不是同一外壳。

## 5. 资产结构

### 5.1 主体与独立模块

- `BLD_QS_M_A_INN_BODY`：墙体、主梁柱、固定石基。
- `KIT_QS_ROOF_*`：可替换主屋顶和附属屋顶。
- `KIT_QS_DOOR_*`、`KIT_QS_WINDOW_*`：粗框门窗模块。
- `KIT_QS_EAVE_*`：单层大木托、少量阳台支撑和入口雨棚。
- `PROP_QS_SIGN_*`、`PROP_QS_LANTERN_*`：独立 Actor 蓝图使用的招牌框和灯笼。

大型建筑若超过单图或单次 3D 生成的稳定范围，按主体、塔楼、侧翼、门廊拆分；每个模块拥有独立文件名、JSON、Pivot、尺寸和 Socket。图片内禁止标注，身份只由文件名和 JSON 表达。

### 5.2 文件与参数权威

- Blender 源文件是结构权威。
- 资产 JSON 是尺寸、模块关系、面数预算、材质槽、Socket 和 PCG 规则的参数权威。
- 固定渲染是视觉验收证据，不承担工程尺寸权威。
- UE StaticMesh 和 Actor 蓝图是运行时产物，不反向覆盖 Blender 源文件。

## 6. 三道验收门

### Gate 1：黄金建筑

代理交付一个完整审核包：

- `.blend` 源文件。
- 固定三分之四、正面和 UE 玩家相机模拟三张渲染。
- 128 px 剪影检查。
- 面数、尺寸、材质槽、模块和几何健康报告。

在此之前，代理自行淘汰对称过强、门窗过细、构件过碎或结构不可建模的版本。用户只判断它能否成为建筑族母版。

### Gate 2：建筑族联系表

黄金样板通过后，代理不逐栋打断用户，直接派生并自检至少五种功能建筑。交付：

- 同相机、同尺度的五栋建筑联系表。
- 单体三分之四渲染。
- 模块复用矩阵和重复度检查。
- 面数、纹理和材质预算汇总。

用户只判断整体是否具有足够变化且仍属于同一世界。

### Gate 3：UE PCG 四镜头

建筑族通过后，先以低成本正式网格替换 B0R 代理，不等待所有细节完成。固定审核：

- 北门进入。
- 城镇核心。
- 主桥。
- 南码头。

该门只讨论布局、重复、密度、尺度、前中后景和性能。单个门窗的小问题记录回资产队列，不在场景审核中即时重做。

## 7. 默认执行权限与沟通协议

三道门之间，以下动作默认获准，无需逐项询问：

- 建模、拆件、UV、基础材质槽、命名、JSON 更新和非破坏性版本保存。
- 在既定视觉契约内调整歪扭度、门窗尺度、梁柱厚度和细节密度。
- 自行生成、渲染、拒绝和重做内部候选。
- 运行 Blender、脚本和 UE 只读/隔离地图验证。

以下动作仍必须停在门点请求用户决定：

- 改变整体中国水墨卡通方向。
- 新增资产类别或明显扩大城镇范围。
- 覆盖用户手调资产、主地图、角色、相机或 PaperZD 内容。
- 需要付费调用、外部发布、不可逆删除或改变已批准黄金样板的核心语法。

用户可以使用自然语言，也可以使用短指令：`歪+1`、`歪-1`、`窗+1`、`碎-1`、`夸张+1`。代理负责转换为参数并写入决定日志，不要求用户重复解释。

每次门点汇报固定为五项：结果、可查看文件、主要变化、自检失败项、下一步。禁止逐条复述全过程。

## 8. 模型、材质与性能预算

- 黄金中型建筑目标 30k–35k 四边面；最大地标或最大商铺可接近 50k，但必须拆分可复用附件。
- 小型建筑目标 12k–20k；沿街中型建筑目标 20k–30k。
- 主体最多 2 个材质槽；招牌、灯笼和特殊附件使用独立实例材质。
- 建筑纹理默认 2K；最大唯一地标才考虑 4K。优先复用风格化材质实例和可平铺表面。
- UE 使用 StaticMesh、Nanite/LOD、实例化重复附件和 PCG 距离裁剪；不把高频小物直接合并进每栋建筑。
- BSDF 卡渲材质统一在 UE 5.8 建立，Blender 材质只承担分区、预览和导出语义。

## 9. 数据流

```text
已有商铺 + 已确认反馈
        ↓
黄金参数 JSON → Blender 黄金建筑 → 自动几何/渲染检查
        ↓ Gate 1
模块库 + 功能参数 → 五类建筑派生 → 联系表/重复度检查
        ↓ Gate 2
FBX/GLB + JSON → UE 隔离资产目录 → B0R PCG 代理替换
        ↓ Gate 3
正式材质、附件、Paper2D、性能收敛
```

## 10. 自动检查与失败处理

Blender 侧至少检查：非流形、重叠面、负缩放、未应用变换、地面平面、Pivot、材质槽、面数和模块命名。渲染侧检查 128 px 剪影、左右非对称、门窗粗细、柱头层数和细节预算。UE 侧检查尺寸、碰撞、入口方向、PCG 排除区、固定镜头可读性、实例数量和性能统计。

内部候选未通过检查时不提交用户审核。每个 Gate 最多进行两次面向用户的受控修订；第二次仍失败时，代理提交明确的冲突和两个解决选择，不继续随机重抽。

## 11. 完成标准

- 三个 Gate 均有明确的批准记录和对应文件哈希。
- 黄金建筑的视觉规则已落入参数 JSON，而不是只存在于聊天记录。
- 五类建筑在轮廓和功能上明显不同，且共享统一材质和模块语言。
- 大型建筑按模块拆分，门窗、招牌、灯笼和屋檐附件可独立复用。
- B0R 四镜头证明建筑在 PCG 场景中不规整、不空、不过度重复。
- 用户在整个建筑生产阶段只需参加三个门点审核；内部迭代不再逐次打断。
